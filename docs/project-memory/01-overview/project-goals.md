# 项目目标与玩法概览

## 核心目标

- 用最少的交互实现高频率、易上手的街机体验（Timberman 类玩法）
- 适配手机浏览器（H5）：触屏操作为主、竖屏体验优先
- 具备“能玩”的基础闭环：菜单 → 开始 → 失败/结算 → 再来一局/返回菜单
- 支持在线双人同屏对战：两名玩家各自控制一棵树，同局比拼得分与生存

## 玩法与交互

- 操作方式：点击屏幕左右半区（Tap halves）
  - 点击左半区：角色站左侧砍树
  - 点击右半区：角色站右侧砍树
- 判定风格：严格判定（Timberman strict）
  - 如果当前层障碍（树枝）在你选择的一侧，则立即失败
  - 成功砍击：得分 +1，并返还时间（由配置决定）
- 时间条：时间持续流逝，归零即失败

## 模式

### 单人

- 目标：在时间归零前尽可能多得分
- 树枝预览：单人会绘制“未来障碍队列”，方便预判
- 主要实现位置：
  - 状态机与运行时：`src/game/state/singlePlayer.ts`
  - 规则：`src/game/rules/TimberRules.ts`

### 在线双人（同屏两棵树）

- 两棵树分别在屏幕左右两侧
- 两位玩家在各自手机端仍按“屏幕中线”为准点击左右半区进行操作
- 对战规则：同局竞速/独立比拼（互不影响输入与判定），以得分与存活为结果指标
- 服务端权威：
  - server 维护房间与模拟状态，周期 tick，并广播最新状态
  - 客户端只发送输入，渲染收到的状态
- 主要实现位置：
  - 客户端：`src/game/net/OnlineClient.ts`、`src/game/net/protocol.ts`
  - 服务端：`server/src/server.ts`、`server/src/gameSim.ts`、`server/src/rooms.ts`、`server/src/protocol.ts`
