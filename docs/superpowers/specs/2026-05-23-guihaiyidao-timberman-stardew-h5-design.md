# 归海一刀（Timberman-like / 星露谷风）设计稿

## 1. 概述

做一款类似 Timberman 的反应类砍树游戏，整体画风偏“星露谷物语”的田园像素风，在手机浏览器（H5）中可玩。核心体验是极短节奏的左右切换 + 砍击 + 躲障碍，在保证爽快与清晰可读的前提下，用 UI 与反馈营造田园氛围。

本设计稿覆盖：玩法与规则、输入与 UI、资产风格、单人模式、在线双人模式、网络同步与反作弊原则、测试与验收。

## 2. 目标与非目标

### 2.1 目标

- 手机上手快：单手即可操作，输入区域大，误触可控。
- 节奏像 Timberman：严格判定、紧张时间条、连续砍击有“上头”感。
- 视觉像“田园午后”：暖木色、柔和自然绿、像素颗粒与木屑粒子。
- 支持两种玩法：
  - 单人：追分/存活时间挑战。
  - 在线双人：同局竞速对抗（同屏布局展示双方进度）。
- 可扩展：后续能较低成本加皮肤/音效/障碍变化/排行榜。

### 2.2 非目标（本阶段不做）

- 复杂的养成系统（建造、升级数值、背包等）。
- 大地图探索或剧情。
- 离线双人同设备对战。
- 重度社交系统（好友、聊天、观战等）。

## 3. 平台与设备假设

- 平台：H5（移动端浏览器为主）。
- 屏幕方向：横屏为主（锁定横屏体验；不强依赖陀螺）。
- 适配：主流 iOS Safari / Android Chrome。
- 运行：Canvas 渲染（2D），60 FPS 为目标。

## 4. 视觉方向

### 4.1 总体氛围

- 方案：经典田园（stardew-classic）。
- 关键词：柔和自然绿、暖木色、低对比但清晰、轻描边、像素颗粒感。

### 4.2 主题皮肤

- 主题：农夫 + 大树（theme-farm-tree）。
- 主要元素：
  - 树干：大树树干分段（被砍后逐段变短）。
  - 障碍：树枝/蘑菇/鸟巢等从左右突出，作为“同侧即死”的障碍。
  - 角色：农夫（斧头）左右切换站位砍树。

### 4.3 反馈风格（建议）

- 砍击：木屑像素粒子 + 轻微屏幕震动（可在设置关闭）。
- 动画：斧头 2~3 帧“预备/命中/回弹”节奏。
- 音效：木头清脆命中、失败“哎呀/扑通”短促明确。

## 5. 核心玩法（单人）

### 5.1 基本循环

1. 玩家在树干左右两侧站位。
2. 点击左右半屏切换站位并砍击（一次输入 = 一次砍击 + 站位设定）。
3. 树干向下缩短一段；上方的障碍（树枝）向下推进。
4. 若玩家当前站位同侧存在障碍，立即失败。
5. 时间条持续减少；砍击可补充少量时间。
6. 失败结算，展示分数/最佳纪录/重开。

### 5.2 判定与失败（严格）

- 严格判定（timberman-strict）：
  - 砍击后结算站位与障碍的同侧关系。
  - 若同侧有障碍，则立即失败。
- 时间条：
  - 作为主要压力来源，持续下降。
  - 砍击补充时间，但存在上限（避免无限拖）。

### 5.3 得分

- 基础：每次成功砍击 +1 分。
- 可选（在实现阶段再定是否启用）：
  - 连击：连续不失误时加成（例如每 N 连击额外 +k）。
  - 速度奖励：短时间内连续砍击更高分（需谨慎，避免奖励“乱点”）。

## 6. 输入与 UI

### 6.1 输入（单人）

- 操作：点屏幕左右半区（tap-halves）。
  - 点左半屏：角色站到左侧并砍击。
  - 点右半屏：角色站到右侧并砍击。

### 6.2 HUD（经典）

- HUD：经典 HUD（portrait-classic-hud 的信息结构迁移到横屏）。
- 组成：
  - 分数：明显可读。
  - 时间条：线性条，随时间减少，砍击补充。
  - 暂停：单人可用。

### 6.3 横屏布局（单人）

- 单人角色居中，树干居中显示，输入仍以屏幕中线分左右。

## 7. 在线双人模式（互联网 / 同屏布局）

### 7.1 模式概述

- 联机方式：互联网在线，两位玩家各自使用自己的手机/浏览器操作。
- 画面布局：双方都看到同一个“对战画面布局”：
  - 左侧为 P1 的树与角色区域。
  - 右侧为 P2 的树与角色区域。
- 每位玩家的输入方式与单人一致：
  - 以自己屏幕中线为准，点左/右半屏决定自己角色站位并砍击。

### 7.2 胜负规则（同局竞速）

- 类型：同局竞速（net-versus-independent）。
- 规则：
  - 两人同时开始。
  - 各砍各的树，失误（同侧障碍）即淘汰。
  - 胜者判定：
    - 先失败者输；若同时失败则以分数高者胜；若仍相同则平局。
- 显示：
  - 顶部同时显示 P1/P2 分数与各自时间条（或同一行双栏）。

### 7.3 断线与退出

- 在线对战不允许暂停。
- 断线处理（建议）：
  - 短暂断线可给极短重连窗口（例如 3 秒），超时判负。
  - 切到后台/锁屏视为断线。

## 8. 障碍生成与随机性

### 8.1 单人

- 使用伪随机序列生成障碍（树枝左右分布）。
- 约束：
  - 避免连续出现“必死局面”（例如同一层左右同时有障碍）。
  - 保持节奏感：一定概率出现空档，但不能过于稀疏。

### 8.2 联机

- 服务器生成或下发 seed，确保双方看到一致的障碍序列与节奏。
- 每棵树可有独立 seed（P1/P2 各自一套序列），以保证公平且可复现。

## 9. 联机同步原则（H5）

### 9.1 架构原则

- 服务器权威：
  - 客户端只上报输入事件（左/右点击 + 本地时间戳）。
  - 服务器计算结果（位置、是否失败、分数、时间条）并广播状态。
- 客户端预测（可选）：
  - 为降低手感延迟，可本地先演算动画与反馈，再用服务器状态对齐纠正。

### 9.2 最小协议（概念）

- 房间：
  - create/join/leave/ready/start。
- 游戏：
  - input：{ playerId, side, clientTime, seq }
  - state：{ t, p1State, p2State, seed, status }
- 可靠性：
  - 输入消息需要幂等（用 seq 去重）。

### 9.3 反作弊（原则）

- 服务器只认可“输入事件”，并对输入频率做限制。
- 不信任客户端上报的分数与判定结果。
- 记录关键事件用于回放或争议排查（可选）。

## 10. 游戏状态机（高层）

- Boot：加载资源、字体、音效。
- Menu：单人/联机入口、设置。
- SinglePlay：单人对局。
- Matchmaking：创建/加入房间、匹配、准备。
- OnlinePlay：联机对局（同步状态）。
- Result：结算页（重开/再来一局/返回）。

## 11. 资产清单（最小可玩版）

### 11.1 角色与动画

- 农夫 idle（左右各 1）。
- 挥斧（左右各 2~3 帧）。
- 失败（左右各 1~2 帧或统一）。

### 11.2 树与障碍

- 树干分段（至少 3 段循环使用）。
- 树枝（左/右），至少 2 种形态做变化。
- 木屑粒子（小方块/碎片）。

### 11.3 UI

- 分数框、时间条、暂停按钮（单人）。
- 联机双人信息栏（P1/P2、时间条、分数）。
- 结算面板与按钮。

### 11.4 音效

- 砍击命中（1~2 个变体）。
- 失败（1 个）。
- 点击/确认（1 个）。
- 背景环境音（可选，低频循环）。

## 12. 可测项与验收标准

- 输入：
  - 点击左右半屏响应稳定，无明显延迟。
  - 连续快速点击不会触发浏览器滚动/缩放等默认行为。
- 规则：
  - 严格判定符合预期；同侧障碍必死、非同侧必活。
  - 时间条下降与补充可感知且可控。
- 适配：
  - 横屏下 UI 不遮挡主画面；不同分辨率不变形。
  - 低端机型保持流畅（目标 60 FPS，最低 30 FPS）。
- 联机：
  - 两端能进入同一房间并开始对局。
  - 延迟情况下仍保持可玩（状态不同步能收敛）。
  - 断线能明确判负或结束对局。

## 13. 待定参数（在实现前确定）

- 时间条：
  - 初始值、每砍补充值、最大值、最小保底（如有）。
- 得分：
  - 是否引入连击、连击上限、加分曲线。
- 障碍生成：
  - 难度曲线：随分数/时间提升障碍密度与排列。
- 联机细节：
  - 匹配方式（好友房/随机匹配）。
  - 重连窗口大小与断线判定。
